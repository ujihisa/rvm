rvm FAQ & Notes

This page is simply documenting common concepts and questions that have come up on irc.

General

  • rvm is intended to be run as an individual user (not as root, yet)
  • rvm must be included at the bottom of the profile files. Otherwise your settings will clobber it's (example: PATH)
  • In order to set a ruby version as default when you open new shells, pass --default when doing the 'rvm use' command.
  • Currently spaces in paths is not supported. Patches welcome :)

Defaults

  • The last ruby version 'used' becomes the default.
  • The default implementation (interpreter) is 'ruby'
  • The default version of ruby is '1.8.6'

Gems

  • Ruby Gems for 1.9.*, jruby, ree are all separate, while with 1.8.X they are shared.

Installation

  • *Everything* is installed into ~/.rvm. To remove rvm from your system simply remove ~.rvm.
  • The gem is only used for distribution, you can just as easily clone the github repo and run bash/rvm-install or bash/rvm-update. After installation the Gem is no longer necessary and can be safely removed. This also means that you can clone the repository locally and symlink ~/.rvm/bin/rvm to the cloned project bash/rvm file.

Mac OS X

  • If you have an error when compiling pertaining to readline, you may need to attempt installing with the following configure option:
    rvm install 1.9 -C --enable-shared,--with-readline-dir=/usr/local Note that this error seems to occur when having a macports installed readline along your path.
